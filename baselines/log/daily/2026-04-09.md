### Benchmark Suite Updates
Changes to the `clawbio-bench` framework focused on fixing existing tests and harness logic:
- **Framework changes**:
  - Added `target_rsid` and `target_gene` parameters to `analyze_acmg_correctness` to scope ACMG criteria extraction to a specific variant.
  - Updated vendored gentropy oracle (`scripts/_reference/gentropy_susie_inf.py`) to support null-weighted SuSiE.
  - Updated derivation script (`scripts/derive_finemapping_ground_truth.py`) to re-derive ground truth for fm_18, fm_20, fm_21.
- **Fixes**:
  - Fixed `_phenotype_matches` to correctly handle byte-identical phenotype strings by adding a fast path and normalized identity check.
  - Corrected the genotype for rs5030655 in the `phi_patient_identifiers_in_header` test from `CC` to `TT` (the actual reference).
  - Updated finemapping ground truth for fm_18, fm_20, fm_21 to use null-weighted SuSiE, aligning with ClawBio's method.
  - Fixed the CVR-ACMG harness to scope criteria to the target variant, preventing false aggregation across the panel.
  - Routed four CVR-ACMG tests that assert unsupported evidence to an advisory category (`criteria_not_machine_parseable`) instead of failing as critical.
- **Coverage impact**: The suite-level pass rate improved from 90.3% (158/175) to 93.1% (163/175), reducing false positives and increasing the reliability of the benchmark.

## Coverage Investigation

**Coverage gaps**: All ClawBio skills explicitly described in the README (bio-orchestrator, pharmgx-reporter, equity-scorer, nutrigx-advisor, claw-metagenomics, clawbio-finemapping, clinical-variant-reporter) have corresponding harnesses. The audit also includes two additional harnesses (`cvr-variant-identity`, `cvr-acmg-correctness`) that are part of the clinical-variant-reporter skill expansion noted in the ROADMAP.

**Test case updates needed**: The `bio-orchestrator` test `inj_03_flock_routing_hijack` shows a critical `unroutable_crash` where no skill was detected, indicating a regression in prompt-injection handling. The `pharmgx-reporter` test `tpmt_compound_het` shows an `incorrect_indeterminate` failure, suggesting changed behavior in compound heterozygote phasing. The `clawbio-finemapping` test `fm_20_susie_inf_est_tausq_activation` shows a critical `susie_inf_est_tausq_ignored` failure, indicating a parameter is silently ignored.

**New patterns**: The `cvr-variant-identity` harness shows a consistent `transcript_selection_error` pattern (3 failures) where unversioned transcript accessions are used, violating HGVS v21.1. The `cvr-acmg-correctness` harness shows `criteria_not_machine_parseable` as an advisory pattern (4 cases), indicating output format issues that hinder automated validation.

**ROADMAP adjustments**: The clinical-variant-reporter Phase 2 work (variant identity, ACMG correctness) is already partially implemented but shows critical failures, suggesting these should be prioritized for remediation before new harnesses. The P1 clinical safety items (variant-annotation, clinpgx, gwas-prs) remain important but current findings indicate existing clinical harnesses need stabilization first.
