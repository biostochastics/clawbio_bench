## clawbio-bench Changes

**Framework changes:**
- Added daily automated audit workflow (`.github/workflows/daily-audit.yml`) with cron scheduling, manual dispatch, and support for historical audits via `clawbio_ref` and `clawbio_baseline_ref` inputs.
- Added `scripts/update_baseline.py` for baseline management (promotes on strict improvement, initializes on first run, handles corrupt baselines).
- Enhanced `scripts/post_summary.py` with multi-model LLM swarm for report synthesis, including daily consolidated reports (`--log-dir`), ClawBio git diff analysis (`--clawbio-repo`), and weekly/monthly trend summaries.
- Added `[all]` optional extra for CI, combining `viz`, `ui`, `finemapping`, and `scikit-learn`.
- Added `--tagged-commits` CLI mode to run benchmarks against only tagged (release/milestone) commits.
- Added `get_tagged_commits()` and `get_commit_tags()` helper functions in core for resolving tagged commits and mapping SHAs to tag names.
- Added delta comparison in Typst reports via `--baseline` CLI flag on `generate_report.py`, showing new/resolved/unchanged findings.
- Implemented unified 5-tier severity system (Pass, Advisory, Warning, Critical, Infra) for all verdict categories, with canonical mapping in `SEVERITY_TIERS` and `TIER_DEFS` in `generate_report.py`.
- Added bento-grid executive dashboard on page 2 of PDF reports with suite status, blocking harnesses, persistent failures, top failure classes, per-harness pass rates, and audit target metadata.
- Replaced single-column verdict list with two-column pass/findings verdict matrix in reports, using colored heatmap squares and grouped by gene/module.
- Added severity indicators in markdown report summary table.

**Coverage impact:**
- Daily automated audits increase reliability and frequency of regression detection.
- Baseline management and delta comparisons improve tracking of changes over time.
- Unified severity system and enhanced reporting (heatmaps, dashboards) improve clarity and actionability of audit results.

A dominant `disclosure_failure` pattern in pharmgx-reporter (9/14 failures) reveals a systemic honesty gap where scope limitations are not communicated. The `stub_silent` pattern in bio-orchestrator shows a safety/honesty gap where stub tools are invoked without warning. The `pip_value_incorrect` pattern in clawbio-finemapping (3 cases) indicates numerical correctness issues in SuSiE PIP calculations.
