## clawbio-bench Changes

Changes to the clawbio-bench suite (version 0.1.4) include:

### Fixes
-   **`_phenotype_matches` function**: Added a byte-equality fast path and a whitespace-normalized identity check to prevent misclassifying identical phenotype strings (e.g., for `mthfr_677tt_methotrexate`).
-   **`phi_patient_identifiers_in_header` test**: Corrected the invalid `rs5030655` genotype in the test input from `CC` to `TT` to match the actual reference allele, fixing a ground-truth mismatch.
-   **Finemapping test cases (`fm_18`, `fm_20`, `fm_21`)**: Updated the vendored gentropy oracle and test input files to include the `null_weight` parameter, aligning the benchmark's ground truth with ClawBio's updated SuSiE implementation.
-   **`analyze_acmg_correctness` harness**: Added `target_rsid` and `target_gene` parameters to scope ACMG criterion extraction to a single variant, preventing incorrect panel-wide aggregation (affects tests like `cvr_24_pvs1_wrong_mechanism` and `cvr_29_benign_combo`).
-   **Four CVR-ACMG tests (`cvr_25`, `cvr_26`, `cvr_30`, `cvr_31`)**: Added a `KNOWN_LIMITATION_DEMO_LACKS_EVIDENCE` flag to route these tests to an advisory category (`criteria_not_machine_parseable`) instead of a critical failure, as they probe evidence not emitted in ClawBio's demo mode.

### Framework changes
-   **ACMG test ground truth files**: Now support `TARGET_RSID:` and `TARGET_GENE:` declarations to specify the variant under test for criteria aggregation.
-   **Advisory verdict handling**: Introduced a `KNOWN_LIMITATION_DEMO_LACKS_EVIDENCE` flag in ground truth to formally mark tests that cannot pass due to missing upstream data, changing their verdict category.

### Coverage impact
-   The suite-level pass rate improved from 90.3% to **93.1%** (163/175 tests) by correcting misclassifications and aligning test logic with ClawBio's behavior.
-   The specificity of ACMG correctness tests is improved by scoping them to individual variants, reducing false positive `classification_aggregation_error` findings.
-   Four tests are now correctly categorized as advisory due to known data limitations in the audit target's demo mode, improving the signal-to-noise ratio of critical findings.
