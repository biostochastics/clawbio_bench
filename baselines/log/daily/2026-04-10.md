## clawbio-bench Changes

Based on the provided changelog for clawbio-bench version 0.1.4, the changes are summarized as follows:

### Framework changes
- **`analyze_acmg_correctness` scoping**: Added optional `target_rsid` and `target_gene` parameters to restrict ACMG criteria analysis to a specific variant, preventing incorrect panel-wide aggregation.
- **Advisory routing for demo limitations**: Introduced a `KNOWN_LIMITATION_DEMO_LACKS_EVIDENCE` ground-truth flag and harness logic to route tests requiring unsupported demo evidence to the `criteria_not_machine_parseable` advisory category.

### Fixes
- **`_phenotype_matches` identity check**: Added a byte-equality fast path and whitespace-normalized identity check to prevent false negatives for identical phenotype strings (e.g., `mthfr_677tt_methotrexate`).
- **`phi_patient_identifiers_in_header` genotype**: Corrected the rs5030655 input genotype from `CC` to `TT` to match the actual reference allele, fixing a false PHI test failure.
- **Finemapping ground truth alignment**: Updated the vendored `gentropy_susie_inf.py` oracle and test inputs (`fm_18`, `fm_20`, `fm_21`) to use the same `null_weight` parameter as ClawBio, then re-derived ground truth.
- **ACMG criteria aggregation**: Fixed `analyze_acmg_correctness` to scope criteria extraction per variant, correcting false `classification_aggregation_error` verdicts in tests like `cvr_24_pvs1_wrong_mechanism` and `cvr_29_benign_combo`.
- **Demo evidence limitations**: Routed four tests (`cvr_25_pvs1_strength_mod`, `cvr_26_pp3_single_tool`, `cvr_30_vcep_brca1`, `cvr_31_vcep_lynch_mlh1`) to advisory status, as they assert evidence not emitted in ClawBio's demo mode.

### Coverage impact
- These fixes corrected five incorrect bench findings, improving the suite-level pass rate from 90.3% to 93.1% and increasing audit reliability by reducing false positives.
