## clawbio-bench Changes

Based on the changelog for clawbio-bench version 0.1.4, the updates to the benchmark suite are:

*   **Fixes**:
    *   **`_phenotype_matches` function**: Added a byte-equality fast path and a whitespace-normalized identity check to prevent misclassifying identical phenotype strings (e.g., for `mthfr_677tt_methotrexate`).
    *   **`phi_patient_identifiers_in_header` test**: Corrected the input genotype for rs5030655 from `CC` to `TT` to match the actual reference allele, fixing a false negative in the PHI test.
    *   **Finemapping ground truth (fm_18, fm_20, fm_21)**: Updated the vendored gentropy oracle and test case inputs to incorporate a `null_weight` parameter, aligning the bench's ground truth with ClawBio's updated SuSiE implementation.
    *   **CVR-ACMG harness (`analyze_acmg_correctness`)**: Added `target_rsid` and `target_gene` parameters to scope criterion aggregation to a single variant, preventing false `classification_aggregation_error` verdicts on multi-variant panels.
    *   **CVR-ACMG test assertions**: Routed four tests (`cvr_25`, `cvr_26`, `cvr_30`, `cvr_31`) that probe evidence not emitted in ClawBio's demo mode to the advisory `criteria_not_machine_parseable` category via a new `KNOWN_LIMITATION_DEMO_LACKS_EVIDENCE` ground-truth flag.

*   **Framework changes**:
    *   Introduced a `KNOWN_LIMITATION_DEMO_LACKS_EVIDENCE` flag in ground-truth files to formally mark tests that are advisory due to missing data in the audit target's demo mode.

*   **Coverage impact**:
    *   The suite-level pass rate improved from 90.3% (158/175) to **93.1% (163/175)** by correcting five misclassifying test verdicts. This increases the reliability of the audit results by reducing false positives from the benchmark itself.
