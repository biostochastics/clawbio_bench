## clawbio-bench Changes

- **Fixes**:
  - Added byte-equality fast path and whitespace-normalized identity check to `_phenotype_matches` to prevent false "Wrong" verdicts for identical phenotype strings (e.g., `mthfr_677tt_methotrexate`).
  - Corrected invalid genotype in `phi_patient_identifiers_in_header` test input from `CC` to `TT` for rs5030655 to align with reference alleles.
  - Updated vendored gentropy oracle (`scripts/_reference/gentropy_susie_inf.py`) and finemapping test inputs (`fm_18`, `fm_20`, `fm_21`) to include `null_weight` parameter, re-derived ground truth via `scripts/derive_finemapping_ground_truth.py`.
  - Scoped ACMG criteria aggregation in `analyze_acmg_correctness` to specific variants via new `target_rsid` and `target_gene` parameters, preventing cross-variant contamination in tests like `cvr_24_pvs1_wrong_mechanism` and `cvr_29_benign_combo`.

- **Framework changes**:
  - Added `KNOWN_LIMITATION_DEMO_LACKS_EVIDENCE` ground-truth flag and harness logic to route tests lacking demo-panel evidence (e.g., `cvr_25_pvs1_strength_mod`, `cvr_30_vcep_brca1`) to advisory `criteria_not_machine_parseable` category instead of critical failures.

- **Coverage impact**:
  - Suite-level pass rate improved from 158/175 (90.3%) to 163/175 (93.1%) by eliminating false-positive findings.
  - Four CVR-ACMG tests now correctly report as advisory when ClawBio's demo panel lacks required evidence, reducing noise from known limitations.

The ROADMAP lists several planned P1 harnesses (e.g., `variant-annotation`, `clinpgx`, `gwas-prs`) with no current coverage.

**New patterns**: The `transcript_selection_error` pattern in `cvr-variant-identity` (3 failures) and `gene_disease_context_missing` in `clinical-variant-reporter` reveal systematic gaps in variant identity and clinical context reporting, warranting dedicated test categories.
